通过调用 Ex35 容器的公共接口，对其进行适配，从而实现栈语义
Stack 不是 Container 的子类实现，而是对 Container 的接口约束、重映射
    像 hook 方法是给具体容器实现的，例如 Vector、List
    而 Stack 不是存储结构，只是限制接口使用方式
	- 存储层（hook / Vector）
	- 使用层（Adapter / Stack）

La classe Stack est implémentée comme un adaptateur qui restreint l’interface d’un conteneur générique existant en s’appuyant exclusivement sur ses méthodes publiques, sans intervenir sur les méthodes hook liées à la structure de stockage.

        ┌────────────────────┐
        │    Vector<T>       │
        └────────▲───────────┘
                 │ private inheritance
        ┌────────────────────┐
        │     Stack<T>       │
        ├────────────────────┤
        │ + push()           │
        │ + pop()            │
        │ + top()            │
        │ + size()           │
        │ + empty()          │
        │ + clear()          │
        └────────────────────┘

L’exercice 36 illustre l’adaptation d’un conteneur générique en une pile à interface restreinte à l’aide du patron Adapter, et démontre, via le patron Strategy, la possibilité de changer dynamiquement la structure de stockage sans modifier l’interface de la pile.

L’implémentation de la pile ne consiste pas à définir une nouvelle structure de stockage via les méthodes hook du conteneur abstrait, mais à adapter l’interface d’un conteneur existant en restreignant son usage aux opérations de type LIFO, en s’appuyant uniquement sur ses méthodes publiques.
